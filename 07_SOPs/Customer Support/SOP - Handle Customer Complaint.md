# SOP - Handle Customer Complaint

## Trigger

Customer submits complaint via any channel (phone, email, app, social media).

## Preconditions

- Customer account verified
- Complaint details provided
- Order reference available (if applicable)

## Steps

### Step 1: Acknowledge Complaint

```
1.1 Thank customer for bringing to attention
1.2 Log complaint with timestamp
1.3 Assign complaint ID
1.4 Set customer expectation for resolution
```

### Step 2: Gather Information

```
2.1 Collect order details (if applicable)
2.2 Document complaint specifics
2.3 Record customer expectations
2.4 Identify required stakeholders
```

### Step 3: Investigate

```
3.1 Review order timeline
3.2 Check system records
3.3 Contact relevant teams (dispatch, driver)
3.4 Identify root cause
```

### Step 4: Propose Resolution

```
4.1 Prepare resolution options
4.2 Get approval if above threshold
4.3 Present resolution to customer
4.4 Document customer acceptance
```

### Step 5: Execute and Follow Up

```
5.1 Implement agreed resolution
5.2 Monitor customer confirmation
5.3 Document final outcome
5.4 Send satisfaction survey
5.5 Log learnings for improvement
```

## Resolution Options

| Issue Type | Standard Resolution |
|-----------|-------------------|
| Delay | Partial refund or credit |
| Damage | Full refund or replacement |
| Service | Apology + service credit |
| Billing | Correction + refund if overcharged |
| Lost cargo | Full compensation + investigation |

## Escalation Triggers

- Compensation > ₹5,000 → Manager approval
- Legal threat → Legal team
- Media/social risk → Communications team
- Repeated complaint → Quality team
