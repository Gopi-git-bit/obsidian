# SOP - Handle POD Disputes

## Trigger

Customer disputes delivery - claims not received, POD not valid, or refuses to acknowledge delivery.

## Preconditions

- Order marked as delivered
- POD uploaded in system
- Customer dispute raised

## Steps

### Step 1: Review POD

```
1.1 Check POD quality and completeness
1.2 Verify recipient signature
1.3 Review timestamp and location
1.4 Check photo evidence if available
```

### Step 2: Investigate

```
2.1 Contact driver for details
2.2 Verify delivery location matches address
2.3 Check for customer communication
2.4 Review delivery attempt records
```

### Step 3: Customer Engagement

```
3.1 Contact customer professionally
3.2 Share POD evidence
3.3 Listen to customer concern
3.4 Document customer claim
```

### Step 4: Resolution

```
4.1 IF valid dispute: Re-deliver or refund
4.2 IF invalid: Provide evidence clearly
4.3 IF partial: Negotiate resolution
4.4 Document decision and reasoning
```

### Step 5: Prevent Recurrence

```
5.1 Identify process gap
5.2 Update SOP if needed
5.3 Train driver on POD best practices
5.4 Flag for quality review
```

## Evidence Checklist

| Required | Description |
|----------|-------------|
| Signature | Recipient name and sign |
| Photo | Delivery location |
| Timestamp | Date and time |
| Location | GPS coordinates |
